canShowTouchId

- (BOOL) canShowTouchId;
Discussion

Method returns touch id can be shown Touch id can only be shown on device that supports it and the passcode has been entered manually since app was launched

- (void)createPasscodeConfirmed:(NSString *)newPasscode;
Parameters
newPasscode

The created passcode, which will be stored and managed by the SDK.

Discussion

Method to be called after the creation of the passcode is confirmed to be successful.

decrementPasscodeAttempts

- (BOOL)decrementPasscodeAttempts;
Return Value

YES if there are remaining attempts available to validate the passcode. NO otherwise.

Discussion

Method to be called after an attempt to validate the passcode has failed, decrementing the number of attempts, and initiating the ultimate failure logic of validatePasscodeFailed if no attempts remain.

showTouchId

- (void) showTouchId;
Discussion

Method to bring up touch id to authenticate device owner If successful, the app will be unlocked Will do nothing if touch id is supported on the device or the passcode has never been entered manually since app was launched

validatePasscodeConfirmed:

- (void)validatePasscodeConfirmed:(NSString *)validPasscode;
Parameters
validPasscode

The successfully validated passcode.

Discussion

Method to be called after passcode validation is confirmed to be successful. This will reset the passcode logic, unlock the screen, and send the app into its subsequent unlocked behavior.

validatePasscodeFailed

- (void)validatePasscodeFailed;
Discussion

Method to be called if all allowed attempts to validate the passcode have failed. The method decrementPasscodeAttempts will call this method automatically, so inheriting classes do not need to do so.
